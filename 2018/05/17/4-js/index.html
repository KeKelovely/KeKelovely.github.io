JavaScript高级程序设计

第21章 Ajax与Comet

  1. Ajax = Asynchronous JavaScipt + XML
  2. 核心是XMLHttpRequest对象，简称XHR
  3. 好处是能够向服务器请求额外的数据而无需卸载页面

    XMLHttpRequest对象

  4. 老版本IE中可能存在三种不同版本的XHR对象：MSXML2.XMLHttp、MSXML2.XMLHttp.3.0、MSXML2.XMLHttp.6.0
  5. IE7及其他浏览器中均支持原生的XHR对象

XHR的用法

  1. 调用和发送请求

  2. 收到响应后，响应的数据会自动填充XHR对象的属性

  • responseText
  • responseXML
  • status
  • statusText
  1. 收到响应前可以使用abort()方法取消异步请求

HTTP头部信息

  1. setRequestHeader()方法可以设置自定义的请求头部信息，接收两个参数，头部字段的名称和头部字段的值

  2. 响应头部信息的取得

GET请求

  1. GET是最常见的请求类型，用于向服务器查询某些信息

POST请求

  1. 使用频率仅次于GET的是POST请求，用于向服务器发送应该被保存的数据

XHR2级

FormData

  1. FormData为序列化表单以及创建与表单格式相同的数据（用于通过XHR传输）提供了便利

  2. 或者用表单元素的数据预先向其中填入键值对

  3. 创建FormData实例后，可以将它直接传给XHR的send()方法

超时设定

  1. IE8为XHR对象添加了一个timeout属性，表示请求在等待响应多少毫秒之后就终止。如果规定时间内服务器没有接收到响应，那么就触发timeout事件

overrideMimeType()方法

  1. Firefox等浏览器引入了overrideMimeType()方法，用于重写XHR响应的MIME类型

进度事件

  • loadstart
  • progress
  • error
  • abort
  • load
  • loadend
  1. 每个请求都从触发loadstart事件开始，接下来是一个或多个progress事件，然后触发error/abort/load事件中的一个，最后以触发loadend事件结束

    load事件

progress事件

  1. onprogress事件处理程序会接收到一个event对象，其target属性是XHR对象，但包含额外的三个属性
  • lengthComputable：进度信息是否可用的布尔值
  • position：已经接收的字节数
  • totalSize：根据Content-Length响应头部确定的预期字节数

跨域资源共享

  1. CORS（Cross-Origin Resouce Sharing，跨域资源共享）定义了在必须访问跨域资源时，浏览器与服务器应该如何沟通。即使用自定义的HTTP头部，决定请求或响应应该成功还是失败。

  2. 在发送请求时，需要附加一个额外的Origin头部

  3. 如果服务器接收这个请求，就在Access-Control-Allow-Origin头部中回发相同信息：

IE对CORS的实现

带凭据的请求

  1. 默认情况下，跨域请求不提供凭据（cookie，HTTP认证以及客户端SSL证明）
  2. 通过将withCredentials属性设置为true，可以指定某个请求应该发送凭据。

    跨浏览器的CORS

  3. 检查是否存在withCredentials属性
  4. 再结合检测XDomainRequest对象是否存在

    图像ping

  5. 与服务器进行简单、单向跨域通信，常用于跟踪用户点击页面或动态广告曝光次数

JSONP

  1. JSON with padding，由回调函数和数据组成。

Comet

  1. Comet是一种服务器向页面推送数据的技术
  2. 长轮询实现
  3. HTTP流实现

    服务器发送事件

  4. SSE = Server-Sent Event是围绕只读Comet交互推出的API或模式

  • SSE API:要预定新的事件流，首先要创建一个新的EventSource对象

  • 事件流：服务器事件会通过一个持久的HTTP响应发送，这个响应的MIME类型为text/event-stream

    Web Sockets

  1. 在一个单独的持久连接上提供全双工、双向通信，建立的连接会从HTTP协议交换为Web Sockets协议

    小结

  1. Ajax是无需刷新页面就能从服务器取得数据的一种方法
  • 核心对象是XHR-XMLHttpRequest对象
  • 同源策略是对XHR的一个主要约束，除非使用CORS或IE中的XDomainRequest对象实现跨域资源访问
  1. Comet是对Ajax的进一步扩展，让服务器几乎能实时向客户端推送数据，实现Comet可通过长轮询和HTTP流做到，SSE是一种实现Comet交互的浏览器AIP，支持长轮询，也支持HTTP流
  2. Web Sockets是一种与服务器进行全双工、双向通信的信道

JavaScript高级程序设计

第22章 高级技巧 小结

  1. 创建作用域安全的构造函数，确保在缺少new操作符时调用构造函数不会改变错误的环境对象
  • 使用惰性载入函数，将任何代码分支推迟到第一次调用函数的时候
  • 函数绑定可让你创建始终在指定环境中运行的函数，同时函数柯里化让你创建已经填写了某些参数的函数
  • 绑定与柯里化结合，允许你在一种任意环境中以任意参数执行任意函数
  1. ECMAScript 5 允许通过以下方式来创建防篡改对象
  • 不可扩展的对象：不允许添加新的属性或方法
  • 密封的对象：不允许删除已有的属性和方法
  • 冻结的对象，不允许重写对象成员
  1. js中可以用setTimeout()和setInterval()创建定时器
  • 定时器代码放在一个等待区域，待时间间隔到了之后，将代码添加到js的处理队列中，等待下一次js进程空闲时被执行
  • 每次一段代码执行结束之后，都会有一小段空闲时间进行其他浏览器处理
  • 可以用定时器将长时间运行的脚本分为一小块可以在以后运行的代码段，这种做法有助于Web应用对用户交互有更积极的响应
  1. js中经常以事件的形式应用观察者模式，虽然事件常常和DOM一起使用，但你也可通过自定义事件在自己的代码中应用。
  2. 拖放在js中可以用鼠标事件和一些简单的计算来实现。

    第23章 离线应用与客户端存储

    数据存储

  3. 目的：属于某个特定用户的信息应该直接存在于该用户的机器上。时至今日，cookie只是在客户端存储数据的其中一种选项。

    Cookie

  4. 服务器对任意HTTP请求发送Set-Cookie HTTP头作为响应的一部分，其中包含会话信息

  5. Cookie的构成

  • 名称：必须URL编码
  • 值：必须URL编码
  • 域
  • 路径
  • 失效时间
  • 安全标志：指定后，只有使用SSL链接时才能发送到服务器，即只能发送到https://www.wrox.com，而不能发送到http://www.wrox.com

  1. 在JavaScript中处理Cookie值时，必须经过URL解码

  2. cookie的性质和它的局限使得其并不能作为存储大量信息的理想手段，所以又出现了其他方法。

    IE用户数据

    Web存储机制

  3. Storage类型
  4. sessionStorage对象
  5. globalStorage对象
  6. localStorage对象
  7. storage事件
  8. 限制

    IndexedDB

  9. 是在浏览器中保存结构化数据的一种数据库，思想是创建一套API，方便保存和读取JavaScript对象，同时还支持查询及搜索
  • 数据库
  • 对象存储空间
  • 事务
  • 使用游标查询
  • 键范围
  • 设定游标方向
  • 索引
  • 并发问题
  • 限制

    小结

  1. 本书讨论了客户端存储的以下内容
  • 以前，这种存储只能用cookie完成。cookie是一小块可以在客户端也可以在服务器端设置的信息，每次发起请求时都会传送它
  • 在js中通过document.cookie可以访问cookie
  • cookie的限制使其可以存储少量数据，然而对大量数据效率很低
  1. IE发明了一种叫用户数据的行为，可以应用到页面的某个元素上
  • 该元素可从一个命名数据空间中载入数据，通过getAttribute()，setAttribute()，removeAttribute()访问
  • 数据必须明确使用save()方法保存到命名数据空间中，以便在会话间持久化数据
  1. Web Storage定义了两种用于存储数据的对象：sessionStorage和localStorage，前者严格用于在一个浏览器会话中存储数据，因为数据在浏览器关闭后会立即删除，后者用于跨会话持久化数据并遵循跨域安全策略。
  2. IndexDB是一种类似SQL数据库的结构化数据存储机制。
