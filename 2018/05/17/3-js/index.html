第十二章 DOM2和DOM3

  1. DOM2级规范定义了一些模块，用于增强DOM1级。
  2. DOM2级核心为不同的DOM类型引入了一些与XML命名空间有关的方法。这些变化只在使用XML或XHTML文档时才有用；对于HTML文档没有实际意义。
  3. 除了与XML命名空间有关的方法外，DOM2级核心还定义了以编程方式创建Document实例的方法，也支持创建DocumentType对象
    4. DOM2级样式模块主要针对操作元素的样式信息而开发，其特性为：
  • 每个元素都有一个关联的style对象，可以用来确定和修改行内的样式
  • 要确定某个元素的计算样式（包括应用给它的所有CSS规则），可以使用getComputedStyle()方法
  • IE不支持getComputedStyle()方法，但为所有元素都提供了能够返回相同信息的currentStyle属性
  • 可以通过document.styleSheets集合访问样式表
  • 除IE之外所有浏览器都支持针对样式表的这个接口，IE也为几乎所有相应的DOM功能提供了自己的一套属性和方法
  1. DOM2级遍历和范围模块提供了与DOM结构交互的不同方式：
  • 遍历即使用NodeIterator或TreeWalker对DOM执行深度优先遍历
  • NodeIterator是一个简单的接口，只允许以一个节点的步幅前后移动。而TreeWalker在提供相同功能的同时，还支持在DOM结构的各个方向上移动，包括父节点、同辈节点和子节点等方向
  • 范围是选择DOM结构中特定部分，然后再执行相应操作的一种手段
  • 使用范围选取可以在删除文档中某些部分的同时，保持文档结构的格式良好，或者复制文档中的相应部分。
  • IE8及更早版本不支持DOM2级遍历和范围模块，但它提供了一个专有的文本范围对象，可以用来完成简单的基于文本的范围操作。IE9完全支持DOM遍历

第18章 JavaScript与XML

小结

  1. JS对XML以及相关技术有很大的支持，然而，由于缺乏规范，共同的功能却存在一些不同的实现。DOM2级提供了创建空XML文档的API，但没有涉及解析和序列化。为了解决这个问题，IE采取了下列方式：
  • 用ActiveX对象来支持处理XML，而相同的对象也可以用来构建桌面应用程序。
  • windows携带了MSXML库，JS能够访问这个库
  • 这个库中包含对基本XML解析和序列化的支持，同时也支持Xpath和XSLT等技术
  1. Firefox实现了两个新类型
  • DOMParser类型比较简单，其对象可以将XML字符串解析为DOM文档
  • XMLSerializer类型执行相反的操作，将DOM文档序列化为XML字符串
  1. 其他浏览器也实现了Firefox中相同的类型，这些类型也成为Web开发中的事实标准。
  2. DOM3级引入了一个针对XpathAPI的规范。这些API可以让JS基于DOM文档运行任何Xpath查询，并且能够返回任何数据的结果。IE以自己的方式实现了对Xpath的支持，即selectSingleNode（）和selectNodes（）方法。在DOM文档中能够查找节点或节点集合。
  3. 最后一种相关技术是XSLT，Firefox为此提供了XSLTProcessor类型，其他浏览器陆续实现。IE则针对XSLT提供了自己的方案，一个是简单的transformNode（）方法，另一个是较为复杂的模板/处理器手段。

第19章 E4X

小结

  1. E4X是以ECMA-357标准的形式发布的一个针对ECMAScript的一个扩展。E4X的目的是为操作XML数据提供与标准ECMAScript更相近的语法。
  2. 到2011年底，FireFox还是唯一一个支持E4X的浏览器。

  3. JSON = JavaScript Object Notation

    语法

  4. JSON语法可以表示以下三种类型的值
  • 简单值：不支持undefined
  • 对象：无序键值对
  • 数组
  • 不支持变量、函数、对象实例

    简单值

  1. JSON字符串必须用双引号表示

对象

  1. JSON对象要求给属性加引号，且不用声明变量

20.1.3 数组

解析与序列化

JSON对象

  1. ECMAScript 5对解析JSON的行为进行了规范，定义全局变量JSON
  • stringify():将JavaScript对象序列化为JSON字符串
  • parse():把JSON字符串解析为原生JavaScript值

序列化选项

  1. JSON.stringify()还可以接收另外两个参数，用于指定以不同的方式序列化JavaScript对象

  • 过滤器：可以是数组或函数

  • 字符串缩进

  • toJSON()方法：可以为任何对象添加自定义的toJSON（）方法

解析选项

  1. JSON.parse()方法接收一个还原函数（reviver）起过滤作用
  • reviver返回undefined，则表示从结果中删除相应的键
  • 返回其他值，则将该值插入到结果中

    小结

  1. JSON是一个轻量级的数据格式，可以简化表示复杂数据结构的工作量。JSON使用JavaScript语法的子集表示对象、数组、字符串、数值、布尔值和null，即使XML也能表示同样复杂的数据结果。但JSON没有那么烦琐，而且在JavaScript中使用更便利
  2. ECMAScript 5 定义了一个原生的JSON对象，可以用来将对象序列化为JSON字符串或者将JSON数据解析为JavaScript对象。JSON.stringify()和JSON.parse()方法分别用来实现上述两种功能。这两个方法都有一定的选项，通过它们可以改变过滤或序列化的方式
