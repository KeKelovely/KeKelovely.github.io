React高级指南

深入JSX

JSX语法糖

用户定义组件必须以大写字母开头

  • 当一个React元素类型以小写字母开头，它表示引用一个类似于
    或者的内置组件，并给React.createElement方法传递’div’或’span’字符串
  • 以大写字母开头的类型，类似于，会被编译成React.createElement(Foo)，对应于自定义组件，或者在JavaScript文件中导入组件

JS表达式作为props

  • 可以用{}包裹的JavaScript表达式作为props属性，并传入给React组件

JSX中的children

  • 你可以在开放标签和闭合标签中放入一些内容，这些内容会被传递给一个特殊的props属性，即props.children

  • 你也可以提供多个JSX元素作为children

  • React组件不能返回多个React元素，但是单个JSX表达式可以有多个子元素，因此如果你想要同时渲染多个元素，可以把它们打包放在1个div中

JS表达式作为children

  • js表达式可以和其他类型的子元素混用，这对于字符串模板而言十分有用

Functions作为Children

  • props.children类似于其他的props属性，可以被传入任何数据，甚至自定义组件的props.children的值可以是一个回调函数

被忽略的元素

  • false、null、undefined和true都是有效的children子元素，但是并不会被渲染

使用PropTypes进行类型检查

  • React内置了类型检测的功能，（例如判断输入的是数字还是字符串），要在组件中进行类型检测，你可以赋值propTypes属性

静态类型检查

静态类型检查

  • Flow和TypeScript作为静态类型检查器可以在运行代码之前识别某些类型的问题，注释代码或发现问题。

Flow

在项目中添加Flow

  • 用Yarn或npm安装flow-bin，然后使用yarn/npm run flow init来初始化Flow配置文件
  • 最后将flow添加到package.json的script部分：”script”:{“flow”:”flow”}

从编译过的代码中剥离Flow语法

  • flow使用特殊的语法为类型注释扩展了js语言，但你需要让浏览器在编译时忽略掉
  • 如果你使用create react app，则flow被默认剥离，不需要任何操作
  • 如果单独使用babel，那么需要安装babel-preset-flow，同时在.babelrc文件中配置flow（和react）

运行flow

  • yarn flow / npm run flow
  • 输出：no errors！（说明你的类型检查没有错误）

添加Flow类型注释

  • 在文件顶部添加批注：//@flow，flow只会检查这类文件

flow的具体用法，请参考flow文档

TypeScript

  • TypeScript是由微软开发的编程语言，是JavaScript的一种类型语言，包含它自己的编译器

在一个项目中添加TypeScript

  • yarn add –dev typescript
  • npm install –save-dev typescript
  • 将tsc命令添加到package.json中： “scripts”:{“build”:”tsc”}

配置TypeScript编译器

  • 运行tsc –init命令新建一个tsconfig.json配置文件
  • 在tsconfig.json中进行配置rootDir和outDir路径，然后，编译器就会将rootDir中的typescript代码输出到outDir路径中去
  • 使用yarn build / npm run build命令，运行typescript编译器

在create react app中使用ts

  • create-react-app my-app –scripts-version=react-scripts-ts，注意这是额外添加的一个第三方项目，不是create react app的一部分

不受控组件

回顾：什么是受控组件？

  • 其值由React控制的输入表单元素称为“受控组件”，例如我们将表单的输入值value存储在React的this.state属性中
  • 好处是，渲染表单的React组件可以控制用户在输入之后的行为（例如只允许用户输入小写字母）

###不受控组件：ref的实现

  • 要实现一个不受控组件，你可以使用ref属性来从DOM中获取表单值

  • 实践：如果你希望的是快速开发、不要求代码质量，不受控组件可以在一定程度上减少代码量。否则，你应该使用受控组件。
