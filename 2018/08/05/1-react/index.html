「React官方文档学习笔记」 1. React快速入门

Posted on 2018-08-05

Hello World

JSX简介

  • React通过使用JSX的扩展语法，更容易地在JavaScript中创建UI

JSX（标签）的属性值

  • 使用双引号来指定字符串字面量作为属性值

  • 用花括号嵌入一个JavaScript表达式作为属性值

JSX是React.createElement()的语法糖

  • 下面两个代码完全等价，第二种是React读取并构建VirtualDOM的方式

编码规范

  • 推荐用括号将JSX包裹起来，好处是避免分号自动插入的陷阱
  • JSX要求使用驼峰(camelCase)命名，如class为className, tabindex为tabIndex

元素渲染

  • 使用ReactDOM.render()方法渲染一个React元素到一个root DOM节点中
  • React元素是不可突变（Immutable）的，一旦元素被创建，就不能再修改其子元素或任何属性，除非重新渲染
  • React DOM会将元素及其子元素与之前版本逐一对比，并只对有必要更新的DOM进行更新（部分更新），以达到DOM所需的状态

组件和属性

  • React中有两种组件：函数组件和类组件

一个好的经验准则

  • 如果你UI的一部分需要用多次，或者本身足够复杂，最好的做法是使其成为可复用组件

编码规范

  • 组件名称总是以大写字母开头
  • 组件必须返回一个单独的根元素，有时需要添加一个div元素来包含多个组件
  • 所有React组件必须是纯函数，并禁止修改其自身的props

状态和生命周期

  • 状态和生命周期是类组件的专有特性

正确地使用state状态

  • 不要直接修改state状态，即使你改了，也不会重新渲染一个组件
  • 用setState()更新state

state更新可能是异步的

  • React为了优化性能，有可能会将多个setState()调用合并为一次更新
  • this.props和this.state可能是异步更新，你不能依赖它们的值计算下一个state（状态）
  • 要解决这个问题，应使用另一种setState()形式，它接受一个函数而不是一个对象，前一个状态作为参数，应用更新时的props作为第二个参数：

从上到下，单向的数据流

  • 任何state始终由某个特定组件所有，不能被其他以外的任何组件访问
  • 从该state导出的任何数据和UI只能影响其DOM树中下方的组件
  • 即使如此，所有的组件相互之间都是完全独立的，不受影响

处理事件

  • React事件使用驼峰命名，如onClick
  • JSX传递一个函数而不是字符串，作为事件处理程序
  • React中不能通过返回false来阻止默认行为，而是需要明确调用preventDefault
  • 注意this的指向：在JavaScript中，类方法默认没有绑定的对象，（因为还未实例化），因此需要使用bind来绑定，如this.handeleClick.bind(this)

条件渲染

  • 使用逻辑&&操作符的内联if用法：在JSX中嵌入表达式

  • 使用条件操作符condition ? true : false

  • 通过return null，来隐藏一个组件，不让其渲染输出

  • 从组件的render方法返回null不会影响组件生命周期方法的触发，componentWillUpdate以及componentDidUpdate仍将被调用

列表和键

  • 多组件渲染：例如，创建一个数组集合，并包含到一个ul元素中，再渲染到DOM

  • 应当为列表元素（例如ul元素中的li元素）提供一个key，key是一个你需要包含的特殊字符串属性

为什么要添加key值

  • key帮助React标识哪个项被修改、添加或移除，数组中的每一个元素都应该有一个唯一不变的键来标识
  • key只在数组的上下文中存在意义，而不要在return的渲染部分将key作为属性传入到组件中
  • key必须在同辈之间唯一，但不需要全局唯一
  • key是React的一个内部映射，但不会传递给组件内部，组件也无法读取

表单

  • 在React中允许表单的默认行为，例如提交表单，上传数据等
  • 但是But！也可以将表单中的数据和值交给React中的state来存储，将表单与React结合，这种，其值由React控制的输入表单元素称为受控组件

  • 相应的，不受控组件就是其值由DOM元素来控制，如input元素中的value属性，在react中可以使用ref与refs来引用非受控组件的值，用ref从DOM获得表单值，而不是为每个状态更新编写一个事件处理程序~

  • 设置表单元素的value属性之后，其显示值将由this.state.value决定，以满足React状态的同一数据理念，每次键盘敲击后会执行handleChange方法以更新React状态，显示值也将随着用户的输入改变

  • 对于受控组件来说，每一次状态state的变化都会伴有相关联的处理函数(onChange={function})，这使得可以直接修改或验证用户的输入

select标签

  • 类似的，React并不用selected属性，而是在根select标签中使用一个value属性，这样你只需更新value属性即可

  • 总结：input，textarea和select都可以以类似的方式工作：接受一个value属性以实现受控组件

file input标签

  • 在React中，一个是只读的(read-only)，不能设置值，只能用File API与文件交互

状态提升

  • 通常情况下，同一个数据的变化需要几个不同的组件来反映。我们建议提升共享的状态到它们最近的祖先组件中
  • 在React中，共享state状态，是通过将其移动到需要它的组件的最近的共同祖先组件来实现的，这被称为状态提升
  • 具体来说，可以在祖先组件中编辑处理函数，再以props属性的方式传入给子组件（注意传入的函数的this仍然绑定在当前组件上），交由子组件去处理，不过子组件已经实现了对父组件中state的共享

组合VS继承

  • 使用自定义的props属性，来实现组件的组合搭配。如和等React元素本质上也是对象，所以可以像其他数据一样作为props属性传递使用

  • 在React中，也可以使用组合的方式（组合就是使用props属性传值）来从一个偏通用的组件渲染出一个偏特殊的组件

组合与继承的取舍

  • 在facebook，我们还没有发现任何用例，值得我们建议你用继承层次结构来创建组件（即class extends）
  • 使用props属性和组合已经足够灵活来明确、安全地定制一个组件的外观和行为，组件可以接受任意的props属性，包括原始值，React元素或函数

React的编程思想

  • 步骤1：将UI拆解到组件层次结构中
  • 步骤2：用React构建一个静态版本
  • 步骤3：确定UI state的最小完整表示
  • 步骤4：确定state的位置
  • 步骤5：添加反向数据流
